# API 网关需求规格

## 1. 背景

系统需要为后端 API 提供统一入口，完成调用方识别、接口鉴权、限流、请求转发、日志记录和基础管理。第一阶段目标是实现一个简单、稳定、可扩展的 API 网关，不追求复杂服务治理。

## 2. 角色

| 角色 | 说明 |
| --- | --- |
| 调用方应用 | 通过网关访问后端 API 的外部或内部系统 |
| 网关管理员 | 配置应用、密钥、路由和限流策略的人员 |
| 上游服务 | 实际处理业务请求的后端服务 |
| 运维人员 | 关注健康状态、日志、性能和故障定位 |

## 3. 目标

- 提供统一的 API 请求入口。
- 支持应用接入、密钥管理和启停控制。
- 支持基于签名的请求认证。
- 支持基于 Redis 的限流。
- 支持路由匹配和 HTTP 请求转发。
- 支持请求日志、错误日志和健康检查。
- 提供基础管理 API 以维护应用、路由和策略。

## 4. 非目标

- 不实现完整可视化管理后台。
- 不实现动态服务发现和注册中心。
- 不实现复杂插件市场。
- 不实现计费、账单和租户套餐。
- 不承诺替代 Nginx、Kong、APISIX 等成熟网关。

## 5. 用户故事

### US-001 应用接入

作为网关管理员，我希望创建调用方应用并生成访问密钥，以便不同系统可以被独立识别和管控。

验收标准：

```text
Given 管理员提交合法应用名称
When 创建应用
Then 系统应生成 app_key 和 app_secret，并保存应用状态为启用
```

### US-002 请求鉴权

作为调用方应用，我希望使用签名访问网关，以便网关确认请求来自可信调用方。

验收标准：

```text
Given 应用存在、状态启用、时间戳有效且签名正确
When 调用方请求网关路由
Then 网关应允许请求进入路由匹配流程
```

```text
Given 签名错误或时间戳过期
When 调用方请求网关路由
Then 网关应返回 401 并记录鉴权失败原因
```

### US-003 路由转发

作为调用方应用，我希望通过网关路径访问上游服务，以便不用直接暴露上游地址。

验收标准：

```text
Given 路由规则已启用
When 请求方法和路径匹配路由
Then 网关应将请求转发到对应上游地址并返回上游响应
```

### US-004 限流

作为网关管理员，我希望为应用或接口配置限流，以便保护上游服务。

验收标准：

```text
Given 某应用在限流窗口内请求次数超过阈值
When 再次请求
Then 网关应返回 429，并不再转发到上游服务
```

### US-005 日志审计

作为运维人员，我希望查看请求日志和错误日志，以便排查问题和评估接口使用情况。

验收标准：

```text
Given 请求经过网关处理
When 请求完成或失败
Then 系统应记录 trace_id、app_key、路由、状态码、耗时和错误原因
```

## 6. 业务规则

| 编号 | 规则 |
| --- | --- |
| BR-001 | 应用被禁用后不得通过鉴权。 |
| BR-002 | app_secret 只允许创建或重置时返回一次，数据库中必须加密或哈希存储。 |
| BR-003 | 签名时间戳默认有效窗口为 300 秒。 |
| BR-004 | nonce 在有效窗口内不可重复使用。 |
| BR-005 | 路由必须同时匹配 HTTP 方法和路径。 |
| BR-006 | 禁用路由不得被匹配。 |
| BR-007 | 限流优先级为接口级策略高于应用级策略。 |
| BR-008 | 上游超时应返回稳定错误码，不暴露内部异常堆栈。 |
| BR-009 | 日志必须脱敏敏感 Header 和密钥字段。 |

## 7. 功能清单

| 模块 | 功能 | 优先级 |
| --- | --- | --- |
| 应用管理 | 创建、查询、启停、重置密钥 | P0 |
| 路由管理 | 创建、查询、启停、匹配规则 | P0 |
| 鉴权 | App-Key、时间戳、nonce、签名校验 | P0 |
| 限流 | Redis 计数、限流响应 | P0 |
| 转发 | HTTP 方法、Header、Body 转发 | P0 |
| 日志 | 请求日志、错误日志、耗时统计 | P1 |
| 健康检查 | MySQL、Redis、自身状态 | P1 |
| 管理接口 | 应用、路由、策略 CRUD | P1 |

## 8. 接口草案

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/health` | 健康检查 |
| POST | `/admin/apps` | 创建应用 |
| GET | `/admin/apps` | 应用列表 |
| PATCH | `/admin/apps/{id}/status` | 启停应用 |
| POST | `/admin/apps/{id}/secret/reset` | 重置密钥 |
| POST | `/admin/routes` | 创建路由 |
| GET | `/admin/routes` | 路由列表 |
| PATCH | `/admin/routes/{id}/status` | 启停路由 |
| ANY | `/gateway/{path}` | 网关转发入口 |
